## Focus

- Designing software that is quiet, useful, and easy to maintain
- Turning rough ideas into working prototypes quickly
- Refining developer and user experience until the flow feels natural
- Keeping architecture practical: enough structure to move fast without making the code heavy

## Working Style

```text
Think clearly.
Ship small.
Measure honestly.
Refine continuously.
```

## Principles

| Principle | What it means in practice |
| --- | --- |
| Clarity over cleverness | Names, flows, and APIs should explain themselves. |
| Taste through restraint | Good interfaces remove friction instead of adding spectacle. |
| Strong defaults | The happy path should be obvious, fast, and hard to misuse. |
| Maintainable momentum | Code should make the next change easier, not just today's change possible. |

## Current Direction

I am most interested in work where product thinking and engineering quality meet:
tools for builders, interfaces for complex workflows, and systems that make repeated work feel lighter.
